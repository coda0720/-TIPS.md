# PayPalで「Suno用プロンプトサイトの閲覧権」を販売するまでの手順

## 1. 販売内容

* 商品：7,000件以上のSuno用プロンプトを掲載した限定サイトの閲覧・使用権
* 価格：買い切り3,000円
* 支払い：PayPal
* 提供方法：購入者のメールアドレスにサイト閲覧権を付与
* 初期運営：決済はPayPal、閲覧権の付与は運営者が手動で行う
* 将来：購入確認から閲覧権付与まで自動化する

---

# 全体の仕組み

```text
GitHubの非公開リポジトリ
└─ サイトと7,000件以上のプロンプトを保存

Cloudflare Pages
└─ GitHubのサイトを公開

Cloudflare Access
└─ 登録済みメールアドレスだけ閲覧可能にする

PayPalビジネスアカウント
└─ 3,000円の支払いを受け付ける
```

## 各サービスの役割

| サービス              | 役割                    |
| ----------------- | --------------------- |
| GitHub            | サイト、プロンプト、更新データの保管    |
| Cloudflare Pages  | GitHubの内容をウェブサイトとして表示 |
| Cloudflare Access | 購入者だけがサイトへ入れるようにする    |
| PayPal            | 代金の受け取りと決済記録の管理       |
| Googleフォームなど      | 購入者の閲覧用メールアドレスを受け取る   |

> 7,000件のプロンプトを入れたGitHubリポジトリは、必ず「非公開」にします。Cloudflare Pagesは公開・非公開のGitHubリポジトリの両方に接続できます。

## 法人の場合

# 第2段階：PayPalを利用可能にする

## 手順1：ビジネスアカウントを作る

PayPalで「ビジネスアカウント」を選択して登録します。

入力する主な情報は次のとおりです。

* 正式な氏名または法人名
* メールアドレス
* パスワード
* 事業形態
* 事業内容
* 銀行口座情報
* 販売サイトやサービスの情報

PayPalでは、個人事業主や事業を始めたばかりの人でもビジネスアカウントを開設できます。

## 手順2：メールアドレスを確認する

PayPalから届く確認メールを開き、メールアドレスを認証します。

このメールアドレスには、次の通知が届きます。

* 支払い完了
* 支払い保留
* 返金
* 購入者からの問い合わせ
* PayPalからの本人確認依頼

販売専用のメールアドレスを用意しておくと管理しやすくなります。

## 手順3：事業者情報を登録する

PayPal管理画面で、次の内容を正確に入力します。

* 事業者名
* 事業形態
* 住所
* 電話番号
* 事業内容
* 取り扱う商品

事業内容の例：

```text
デジタルコンテンツおよびAI音楽生成用プロンプトの利用ライセンス販売
```

商品説明の例：

```text
Suno等の音楽生成サービスで使用できるプロンプトを検索・閲覧できる限定ウェブサイトの買い切り利用権
```

## 手順4：本人確認を完了する

PayPal管理画面に表示される案内に従い、本人確認を行います。

主な流れは次のとおりです。

```text
本人情報の確認
↓
本人確認書類の撮影・提出
↓
顔写真などによる本人確認
↓
PayPalによる確認
↓
本人確認完了
```

ビジネスアカウントで支払いの受け取りや残高の引き出しを行うには、本人確認が必要です。

## 手順5：銀行口座を登録する

PayPal管理画面の「ウォレット」から銀行口座を登録します。

```text
PayPalへログイン
↓
ウォレット
↓
銀行口座の登録
↓
銀行名・支店名・口座番号などを入力
```

PayPalの登録名義と銀行口座の名義が異なると、登録や引き出しができない場合があります。

## 手順6：セキュリティを設定する

販売開始前に、次の設定を行います。

* 二段階認証を有効にする
* 推測されにくいパスワードを設定する
* PayPalを装ったメールのリンクを不用意に開かない
* 支払いはメールだけで判断せず、必ずPayPal管理画面で確認する

ここまで完了すれば、運営者がPayPalで代金を受け取る準備は完了です。

# 第3段階：3,000円の商品をPayPalに作る

## ペイメントリンクを作成する

PayPalビジネスアカウントで、ペイメントリンクを作成します。

ペイメントリンクは、ウェブサイトの決済システムを自分で開発しなくても、商品名と価格を設定して支払いを受け付けられるURLです。

## 商品設定例

```text
商品名：
7,000種類以上のSuno用プロンプトライブラリ
買い切り利用ライセンス

価格：
3,000円

数量：
1

配送：
なし

商品形式：
デジタルコンテンツ・サイト閲覧権
```

## 商品説明例

```text
7,000件以上のSuno向け音楽生成プロンプトを、
検索・閲覧・コピーして使用できる限定サイトの買い切り利用権です。

購入後、登録されたメールアドレスに閲覧権を付与します。

プロンプト一覧そのものの転載、再販売、共有、
第三者へのアカウント貸与は禁止します。
```

## 購入完了後の移動先

PayPalでは、購入完了後に購入者を指定ページへ移動させる設定ができます。

移動先は、限定サイトそのものではなく、次のような「購入完了案内ページ」にします。

```text
お支払いありがとうございます。

限定サイトの閲覧権を付与するため、
以下のフォームから必要事項をご送信ください。

・購入者名
・PayPalで使用したメールアドレス
・閲覧に使用するメールアドレス
・PayPal取引ID
```

> 限定サイトのURLを購入完了画面に直接表示しても、閲覧権を登録していない購入者は入れないようにします。

# 第4段階：販売案内ページを作る

GitHub Pagesなどを使い、誰でも見られる販売案内ページを作ります。

## 販売ページに必要な内容

### 1. 商品名

```text
7,000種類以上のSuno用プロンプトライブラリ
買い切り利用ライセンス
```

### 2. 商品説明

* どのようなプロンプトがあるか
* どのように検索できるか
* コピーして使用できること
* 掲載件数
* サンプル
* 対応ジャンル
* 利用方法

### 3. 価格

```text
買い切り3,000円
追加の月額料金なし
```

### 4. 購入後の流れ

```text
PayPalで支払い
↓
購入者情報を送信
↓
運営者が支払いを確認
↓
閲覧用メールアドレスを登録
↓
限定サイトのURLを送信
```

### 5. 権利の範囲

購入者に与えるのは、プロンプトの所有権ではなく「使用許諾」です。

```text
購入者は、掲載されているプロンプトを
Suno等の音楽生成サービスで使用できます。

ただし、プロンプト一覧そのものの転載、複製、
再配布、再販売、共有、データベース化は禁止します。
```

### 6. 更新の扱い

次のどちらかを明記します。

#### 更新分も含める場合

```text
購入後に追加されるプロンプトも、
追加料金なしで閲覧できます。
```

#### 購入時点の内容を保証する場合

```text
本商品は購入時点で掲載されている
7,000件以上のプロンプトを対象とします。

将来追加される別シリーズや大型アップデートは、
対象外となる場合があります。
```

運営負担を考えると、無期限更新を約束せず、次の表現が安全です。

```text
軽微な修正やプロンプト追加は無料で行う場合がありますが、
将来のすべての更新を保証するものではありません。
```

最低限、次のページを作ります。

## 特定商取引法に基づく表記に必要な主な項目

```text
販売事業者名
運営責任者
所在地
電話番号
メールアドレス
販売価格
販売価格以外に必要な費用
```

## 返金条件の例

```text
デジタルコンテンツおよび閲覧権という商品の性質上、
閲覧権の付与後は、原則として購入者都合による
返品・返金には対応していません。

ただし、二重決済、閲覧権が提供されない場合、
その他運営者側の不備が確認された場合は、
```

# 第7段階：購入者だけ閲覧可能にする

Cloudflare Accessを設定し、許可されたメールアドレスだけログインできるようにします。

## ログイン方法

```text
購入者が限定サイトを開く
↓
メールアドレスを入力
↓
メールに一時認証コードが届く
↓
コードを入力
↓
限定サイトへ入る
```

Cloudflare Accessでは、運営者が許可したメールアドレスにだけ一時認証コードを送る設定ができます。認証コードは一度だけ使用でき、発行から10分で期限切れになります。

## 購入者を追加する方法

```text
Cloudflare Zero Trust
↓
Access
↓
Applications
↓
限定サイトを選択
↓
Policies
↓
Allowルール
↓
購入者のメールアドレスを追加
```

## 注意

One-time PINを有効にするだけでは、誰でも入れる設定になる可能性があります。

必ず次の条件を組み合わせます。

```text
ログイン方法：One-time PIN
＋
許可条件：購入者のメールアドレス
```

# 第8段階：実際の販売・運営手順

## 購入者側

```text
1. 販売案内ページを見る
2. 利用規約と返金条件を確認する
3. PayPalで3,000円を支払う
4. 購入完了後のフォームを送信する
5. 運営者から限定サイトの案内を受け取る
6. メール認証コードでログインする
7. プロンプトを検索・コピーして使用する
```

## 運営者側

```text
1. PayPalから支払い通知を受け取る
2. PayPal管理画面を開く
3. 支払い状態が「完了」になっていることを確認する
4. 購入者が送信した情報と照合する
5. Cloudflare Accessへ閲覧用メールアドレスを追加する
6. 限定サイトのURLを購入者へ送る
7. 購入者一覧に登録日とPayPal取引IDを記録する
```

## 支払い確認で見る項目

* PayPal取引ID
* 購入者名
* 購入者メールアドレス
* 支払金額
* 通貨
* 支払い日時
* 支払い状態
* 返金・異議申し立ての有無

> PayPalから届いたメールだけで判断せず、必ずPayPal公式管理画面にログインして確認します。

# 費用

## PayPal

* アカウント開設：無料
* 初期費用：無料
* 月額費用：基本無料
* ペイメントリンク作成：無料
* 売れた時に決済手数料が発生

2026年1月22日更新のPayPal料金では、日本国内の標準商用取引は次の手数料です。

```text
販売価格の3.6％＋40円
```

3,000円で販売した場合：

```text
商品価格：3,000円
割合手数料：108円
固定手数料：40円
合計手数料：約148円
PayPal残高：約2,852円
```

海外取引では、国内取引の手数料に0.5％が追加され、通貨換算が発生する場合は別途換算費用も考慮する必要があります。

## 銀行口座への引き出し

PayPal残高を日本の銀行口座へ引き出す場合、現在の料金は次のとおりです。

```text
1回50,000円以上：手数料無料
1回50,000円未満：手数料250円
```

そのため、少額販売ごとに引き出すより、PayPal残高が5万円以上になってからまとめて引き出す方が、手数料を抑えられます。

## その他

* GitHub：無料プランから開始可能
* Cloudflare Pages：無料プランから開始可能
* Cloudflare Zero Trust：無料プランから開始可能
* 独自ドメイン：任意
* Googleフォーム：無料で利用可能

独自ドメインを購入せず、各サービスの無料枠を使う場合、固定運営費をかけずに開始できます。ただし、各サービスの無料枠や仕様は将来変更される可能性があります。Cloudflareには無料プランが用意されています。

この商品が許諾できるのは「プロンプトの使用権」です。

生成した楽曲の権利や商用利用について、販売者が一律に保証することは避けます。

```text
本商品は、音楽生成用プロンプトの使用許諾を提供するものです。

生成された楽曲の権利、商用利用、配信、収益化などの条件は、
購入者が利用するSunoの契約プランおよび

本商品はSuno公式商品ではなく、
Sunoとの提携・協力・公認を示すものではありません。
```

Sunoの規約は変更される可能性があり、入力内容に必要な権利を持っていることも利用者側に求められています。

# 主な懸念点

## 1. コピーや転載を完全には防げない

購入者がサイトを閲覧できる以上、次の行為を技術的に完全に防ぐことはできません。

* コピー
* スクリーンショット
* 手動保存
* 第三者への転載
* プロンプトの共有

対策として、次を組み合わせます。

* 利用規約を設ける
* メールアドレスごとにログインさせる
* 共通パスワードを使わない
* ログイン履歴を確認する
* 一覧を一度に大量表示しない
* 不正利用者のアクセスを停止できるようにする

## 2. PayPalとCloudflareは自動連携しない

ペイメントリンクだけでは、PayPalがCloudflareへ購入者を自動登録してくれるわけではありません。

初期運営では、支払い確認後に運営者が購入者のメールアドレスを登録します。

## 3. 返金した購入者の閲覧権を削除する必要がある

返金、チャージバック、決済取消しが発生した場合は、Cloudflare Accessから該当メールアドレスを削除します。

## 4. 「永久利用」を約束しすぎない

買い切りは「一度支払えば毎月の料金が発生しない」という意味にします。

次のような無期限保証は避けます。

```text
サイトを永久に維持します
必ず一生利用できます
今後の全プロンプトを永久に追加します
```

代わりに次のように記載します。

```text
本商品は月額料金の発生しない買い切り商品です。

サービスの重大な仕様変更、法令変更、
外部サービスの終了、災害その他の事情により、
提供方法を変更または終了する場合があります。
```

* PayPal Webhook
* Cloudflare Workers

* 購入者管理画面

ただし、販売開始時点から自動化する必要はありません。

最初は手動で販売を開始し、購入者数や問い合わせ内容を確認してから自動化した方が、構築ミスや不要な開発を減らせます。

---

# 最短で販売を開始する順番

```text
1. 販売用メールアドレスを作る
2. PayPalビジネスアカウントを作る
3. 本人確認を完了する
4. 銀行口座を登録する
5. 3,000円のペイメントリンクを作る
6. GitHubに販売案内ページを作る
7. 利用規約・返金規定・特商法表記を作る
8. プロンプト用GitHubリポジトリを非公開にする
9. Cloudflare Pagesへ接続する
10. Cloudflare Accessを設定する
11. 購入者情報送信フォームを作る
12. 決済・ログイン・検索・コピーをテストする
13. 販売ページを公開する
14. 支払い確認後、購入者を手動登録する
```

---

# 完成後の運営形態

```text
固定運営費：
無料枠を使えば基本0円

販売ごとの費用：
PayPal決済手数料

運営者の主な作業：
支払い確認
購入者メールアドレスの登録
案内メールの送信
問い合わせ対応
返金時のアクセス削除

購入者の作業：
PayPalで支払う
閲覧用メールアドレスを送る
メール認証コードでログインする
```

## 最終結論

GitHubだけでは、購入者限定サイトとPayPal決済を安全に連携できません。

最短かつ固定費を抑えた構成は、次の組み合わせです。

```text
GitHub
＋
Cloudflare Pages
＋
Cloudflare Access
＋
PayPalペイメントリンク
```

最初は閲覧権の付与だけ手動にすることで、複雑なプログラミングを行わずに、買い切り3,000円のプロンプト閲覧権を販売できます。
