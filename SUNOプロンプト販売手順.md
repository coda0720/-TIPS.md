# SUNO AIプロンプト閲覧サイトを買い切り3,000円で販売する方法

## 目的

販売するものは、サイトそのものではありません。

購入者が一度3,000円を支払うと、会員専用サイトに入り、掲載されているSUNO AI用プロンプトを自由に検索・閲覧・コピーできるサービスです。

```text
購入者が3,000円を支払う
↓
運営者が支払いを確認する
↓
購入者のメールアドレスを登録する
↓
購入者がメール認証でログインする
↓
プロンプトを検索・閲覧・コピーする
```

---

# 使用するサービス

| サービス              | 役割                |           費用 |
| ----------------- | ----------------- | -----------: |
| GitHub            | サイトのデータを保存する      |           無料 |
| Cloudflare Pages  | サイトをインターネット上に公開する |        無料枠あり |
| Cloudflare Access | 購入者だけサイトに入れるようにする |   小規模なら無料枠あり |
| PayPal            | 3,000円の代金を受け取る    | 月額無料・決済手数料あり |
| Googleスプレッドシート    | 購入者一覧を管理する        |           無料 |

毎月の固定費を0円に近づけられます。

ただし、商品が売れたときにはPayPalの決済手数料が発生します。

---

## Cloudflare Pages

GitHubに保存しているサイトを、インターネット上で公開するサービスです。

公開すると、次のようなURLが発行されます。

```text
https://サイト名.pages.dev
```

---

## Cloudflare Access

会員専用サイトの入口に置く受付機能です。

登録されたメールアドレスだけ、サイトに入れるようにします。

```text
登録済みメールアドレス
→ サイトに入れる

未登録メールアドレス
→ サイトに入れない
```

# 販売するサイトの構成

サイトを次の2つに分けます。

## 1．一般公開ページ

誰でも見られる販売ページです。

掲載する内容は次のとおりです。

* サービスの説明
* プロンプトの一部見本
* 価格
* PayPal支払いボタン
* 利用規約
* プライバシーポリシー
* 特定商取引法に基づく表記
* お問い合わせ先

---

## 2．購入者限定ページ

3,000円を支払った購入者だけが見られるページです。

掲載する機能は次のとおりです。

* プロンプト一覧
* キーワード検索
* ジャンル検索
* タグ検索
* プロンプトのコピー
* プロンプトの使い方
* カテゴリー選択

---

# 売れる状態にする手順

## 手順1：現在のサイトを完成させる

最低限、次の機能が正常に動く状態にします。

* プロンプトを一覧表示できる
* キーワードで検索できる
* ジャンルで絞り込める
* タグで絞り込める
* コピーボタンが動く
* スマートフォンでも表示できる
* エラーが出ない

---

## 手順2：サイトを一般用と会員用に分ける

例として、次のような構成にします。

```text
suno-prompt-site
├─ public
│  ├─ index.html
│  ├─ price.html
│  ├─ terms.html
│  ├─ privacy.html
│  └─ tokushoho.html
│
└─ members
   ├─ index.html
   ├─ style.css
   ├─ script.js
   └─ prompts.js
```

`public`は一般公開部分です。

`members`は購入者限定部分です。

---

## 手順3：GitHubを非公開にする

販売用サイトのGitHubリポジトリは、基本的に非公開にします。

### Public

誰でもサイトのファイルを見ることができます。

### Private

許可された人だけがサイトのファイルを見ることができます。

設定手順は次のとおりです。

```text
GitHubで対象リポジトリを開く
↓
Settings
↓
General
↓
Danger Zone
↓
Change repository visibility
↓
Privateを選択
```

プロンプトがHTMLやJavaScriptに直接書かれている場合、公開リポジトリにすると内容を見られる可能性があります。

---

## 手順4：Cloudflareアカウントを作る

Cloudflareのアカウントを作成します。

その後、GitHubと接続します。

```text
Cloudflareにログイン
↓
Workers & Pages
↓
Create application
↓
Pages
↓
Import an existing Git repository
↓
GitHubのリポジトリを選択
↓
公開する
```

公開すると、次のようなURLが発行されます。

```text
https://サイト名.pages.dev
```

独自ドメインを使わなければ、ドメイン料金は必要ありません。

---

## 手順5：一般公開ページと会員ページを分けて公開する

例として、次のように分けます。

```text
一般公開ページ
https://example.pages.dev

会員専用ページ
https://members.example.pages.dev
```

一般公開ページには、価格や購入方法を表示します。

会員専用ページには、プロンプト一覧を表示します。

---

## 手順6：Cloudflare Accessを設定する

会員専用ページに、メールアドレスによる入場制限を付けます。

```text
Cloudflare
↓
Zero Trust
↓
Access
↓
Applications
↓
Add an application
↓
Self-hosted
↓
会員専用サイトのURLを登録
```

その後、購入者のメールアドレスだけを許可する設定を作ります。

```text
許可メールアドレス
aaa@example.com
bbb@example.com
ccc@example.com
```

この一覧に登録されていない人は、会員サイトへ入れません。

---

## 手順7：メール認証を設定する

Cloudflare Accessで、ワンタイムPINによるログインを使います。

購入者のログイン手順は次のとおりです。

```text
会員サイトを開く
↓
メールアドレスを入力
↓
確認番号がメールに届く
↓
確認番号を入力
↓
プロンプト一覧が表示される
```

全員に同じパスワードを教える必要はありません。

---

## 手順8：PayPalで3,000円の商品を作る

PayPalビジネスアカウントを使用します。

設定例は次のとおりです。

```text
商品名：
SUNO AIプロンプトライブラリー閲覧権

価格：
3,000円

支払い：
1回のみ

内容：
会員専用サイトに掲載されたプロンプトを
検索・閲覧・コピーできる利用権
```

PayPalで支払いリンクを作成し、一般公開ページに設置します。

表示例：

```text
［3,000円で閲覧権を購入する］
```

# 購入後の運営手順

最初は、購入確認と会員登録を手動で行います。

## 購入者側

```text
1．販売ページを開く

2．PayPalで3,000円を支払う

3．運営者から利用開始の連絡を受ける

4．会員サイトを開く

5．メールアドレスを入力する

6．メールに届いた確認番号を入力する

7．プロンプトを検索・閲覧・コピーする
```

---

## 運営者側

```text
1．PayPalの支払い通知を確認する

2．購入者のメールアドレスを確認する

3．Cloudflare Accessを開く

4．購入者のメールアドレスを許可リストへ追加する

5．購入者へ会員サイトのURLを送る
```


# 購入者一覧を管理する

Googleスプレッドシートなどで、購入者を記録します。

| 購入日        | メールアドレス                                       |     金額 | 入場許可 | 備考         |
| ---------- | --------------------------------------------- | -----: | ---- | ---------- |
| 2026/07/03 | [example@gmail.com](mailto:example@gmail.com) | 3,000円 | 許可済み | PayPal確認済み |

記録しておく情報は次のとおりです。

* 購入日
* 購入者名
* メールアドレス
* 支払い金額
* 入場許可の状態
* 返金の有無
* 規約違反の有無

# 利用規約に書く内容

## 許可すること

* プロンプトをコピーする
* プロンプトを自由に変更する
* SUNO AIでの楽曲制作に使用する
* 生成した楽曲を公開する
* 生成した楽曲を商用利用する

生成楽曲の利用については、購入者自身が使用しているSUNO AIのプランや利用規約にも従う必要があります。

---

## 禁止すること

* プロンプト一覧そのものを再販売する
* プロンプトを大量に第三者へ配布する
* サイト内容を他のサイトへ転載する
* 購入権を第三者へ譲渡する
* ログイン用メールアドレスを共有する
* 自動ツールでサイト全体を取得する
* サイトの内容を複製して類似サービスを販売する

# 無料で運営できる範囲

## 固定費0円で始められるもの

* GitHub無料プラン
* Cloudflare Pages無料枠
* Cloudflare Access無料枠
* Cloudflareの無料URL
* Googleスプレッドシート
* PayPalビジネスアカウント

---

## 費用が発生する可能性があるもの

* PayPalの決済手数料
* 独自ドメインを購入する場合の料金
* 無料枠を超えた場合の料金
* 運営者の作業時間

正確には、次の運営方法になります。

```text
毎月の固定費は0円に近い
↓
商品が売れたときだけPayPal手数料が発生
```

---

# この方法のメリット

* 毎月の固定費を抑えられる
* 複雑な会員システムを作らなくてよい
* パスワードを管理しなくてよい
* 購入者ごとに入場を許可できる
* 問題があった購入者だけ入場を停止できる
* GitHubでプロンプトを更新できる
* 購入者は検索・閲覧・コピーだけを簡単に行える

---

# 注意点

正規購入者がプロンプトを閲覧できる以上、次の行為を完全に防ぐことはできません。

* コピー
* 画面保存
* スクリーンショット
* 手作業での転載

そのため、次の2つを組み合わせて運営します。

```text
Cloudflare Access
→ 未購入者をサイトへ入れない

利用規約
→ 購入者による再販売や大量配布を禁止する
```
